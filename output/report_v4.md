## 핵심 코드 로직

### 첨부 추출 - 라인 기반으로 교체 (핵심 변경)

5페이지 실제 구조:
```
첨부1  일본 공개특허공보 특개2023-004524호(2023.01.17.) 1부.
첨부2  일본 공개특허공보 특개2021-042988호(2021.03.18.) 1부.
첨부3  일본 공개특허공보 특개2020-001861호(2020.01.09.) 1부.  끝.
```

라인 기반 파싱:
```python
def extract_attachments(doc):
    for page in doc:
        for line in text.split("\n"):
            line = line.strip()
            if re.match(r"첨부\d+\s+", line):
                value = re.sub(r"^첨부\d+\s+", "", line)
                value = re.sub(r"\s*끝\.?\s*$", "", value).strip()
                attachments.append(value)
    # → 3개 모두 수집
```

## 버전별 개선 히스토리

| 버전 | 주요 변경 | 해결된 문제 |
|---|---|---|
| v1 | 기본 추출 + 폰트 진단 | 표 추출 확인, 섹션/서브섹션 구조 파싱 |

## PDF 구조 분석 정리

| 페이지 | 내용 | 추출 방법 | 비고 |
|---|---|---|---|
| 1 | 표지 + [심사결과] | PyMuPDF + 정규식 | 특수 2컬럼 레이아웃 |
| 2 | [구체적인거절이유] + 비교표 + 1-1 | PyMuPDF + pdfplumber | 표 bbox 기반 중복 제거 |
| 3 | 1-2 ~ 1-4 | PyMuPDF | 텍스트 기반 |
| 4 | 1-5 ~ 1-8 | PyMuPDF | 텍스트 기반 |
| 5 | 1-8 마지막 + [첨부] | PyMuPDF | 첨부 라인 기반 파싱 |
| 6 | 심사관 안내문 | 스킵 | 텍스트 추출은 가능하나 불필요 |

## 정보손실 여부

**정보손실 없음** - 원본 PDF 텍스트 전체가 JSON에 보존되며,
구조(섹션/서브섹션/표)를 추가한 것으로 원본 텍스트 변환 없음.
