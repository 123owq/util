## 1. "OA에 구성요소별 상세 매핑이 있는가"

**좋은 OA (학습에 쓸 수 있음):**

```
청구항 1에 대하여,
인용문헌 1의 [0023]에 "트레드부에 주홈이 형성된 구성"이 개시되어 있고,
인용문헌 1의 [0031]에 "블록에 사이프가 형성된 구성"이 개시되어 있고,
인용문헌 2의 [0045]에 "실리카 30~100 중량부를 포함하는 고무 조성물"이
개시되어 있다.
다만, 인용문헌 1, 2에는 "사이프 깊이가 주홈 깊이의 40~60%"인 
구성은 개시되어 있지 않으나, 이는 설계사항에 불과하다.
```

이걸 파싱하면 구성요소 4개 × 개시 여부 라벨이 깔끔하게 나옵니다. Tool 3 학습 데이터로 바로 쓸 수 있는 거예요.

**나쁜 OA (학습에 못 씀):**

```
청구항 1~5는 인용문헌 1 및 2에 의해 
특허법 제29조 제2항의 규정에 의하여 특허를 받을 수 없다.
```

이건 파싱해봤자 "청구항 1~5가 거절됨"이라는 사실밖에 안 나옵니다. 어떤 구성요소가 어떤 문헌의 어떤 부분에 대응되는지 전혀 모르니까 Tool 3 학습 데이터로 쓸 수 없습니다.

## 2. "공개 청구항 → 등록 청구항 diff가 명확한가"

**좋은 케이스:**

```
[공개 독립항]
트레드부에 복수의 주홈이 형성된 타이어.

[등록 독립항]
트레드부에 복수의 주홈이 형성되고,
상기 주홈 사이의 블록에 사이프가 형성되며,
★ 상기 사이프의 깊이가 상기 주홈 깊이의 40~60%인 것을
특징으로 하는 타이어.
```

diff가 명확합니다. "사이프 깊이 40~60%"가 추가됐다는 걸 자동으로 뽑을 수 있고, 이게 Tool 5의 학습 데이터(보정 정답)가 됩니다.

**나쁜 케이스:**

```
[공개 독립항]
고무 조성물에 있어서, 천연고무 100 중량부에 대하여
실리카를 포함하는 고무 조성물.

[등록 독립항]
타이어 트레드용 고무 조성물에 있어서,
스티렌-부타디엔 고무 및 부타디엔 고무를 포함하는 고무 성분 100 중량부에 대하여,
BET 비표면적이 150~200 m²/g인 실리카 40~80 중량부,
비스(트리에톡시실릴프로필)테트라설피드 4~12 중량부를 포함하며,
상기 고무 조성물의 인장강도가 15 MPa 이상인 것을 특징으로 하는 고무 조성물.
```

청구항이 완전히 다시 작성된 케이스입니다. 뭐가 추가된 건지, 뭐가 수정된 건지 자동 diff가 어렵습니다. 사실상 별개의 청구항이 되어버려서 "어떤 구성요소를 추가해서 극복했는가"를 자동으로 뽑기 힘듭니다.

## 3. "인용 선행문헌 전문을 확보할 수 있는가"

**좋은 케이스:**

```
인용문헌: KR 10-2019-0012345 (한국 공개특허)
→ KIPRIS에서 전문 PDF 바로 다운로드 가능
→ 텍스트 파싱해서 [0023], [0045] 단락을 정확히 추출 가능
```

**나쁜 케이스:**

```
인용문헌: JP 2018-123456 (일본 공개특허)
→ KIPRIS에 없음, J-PlatPat에서 일본어로만 제공
→ 한국어 번역이 필요하고 단락 번호 체계도 다름

인용문헌: "Rubber Chemistry and Technology, Vol.85, pp.123-145 (2012)"
→ 비특허문헌(논문). KIPRIS에 없음
→ 학술 DB에서 별도 확보해야 하고 PDF 파싱도 다름
```

인용문헌이 한국 특허면 KIPRIS에서 바로 가져오지만, 일본/미국 특허나 논문이면 확보 자체가 어렵거나 추가 비용이 듭니다. Tool 3에서 "이 구성요소가 선행문헌 [0045]에 개시되어 있는가"를 판단하려면 그 [0045] 텍스트가 있어야 하는데, 없으면 학습 데이터로 못 씁니다.

## 4. "PDF 파싱이 깨지지 않았는가"

**좋은 케이스 (텍스트 기반 PDF):**

```
파싱 결과:
"실리카 30~80 중량부, 실란커플링제 5~15 중량부를 포함하는"
→ 깔끔하게 추출됨
```

**나쁜 케이스 (파싱 깨짐):**

```
파싱 결과:
"실리카 30∼80 ̊ ̊부, 실란커 ̊링제 5∼15 ̊ ̊부를 포함하는"
→ 특수문자, 위첨자, 화학식이 깨짐

또는 표가 있는 경우:
[실시예 표]
  실시예1  실시예2  비교예1
실리카(중량부)  50     70     20
마모지수       105    110    85

파싱 결과:
"실시예1 실시예2 비교예1 실리카(중량부) 50 70 20 마모지수 105 110 85"
→ 행/열 구조가 무너져서 어떤 숫자가 어떤 실시예인지 구분 불가
```

이게 특히 타이어 특허에서 치명적입니다. 고무 조성물 특허는 거의 반드시 실시예/비교예 표가 있는데, 이 표가 깨지면 Tool 3에서 "선행문헌의 실시예 범위와 청구항 수치 범위를 비교"하는 게 불가능합니다.

## 50건 파일럿의 구체적 실행

50건을 실제로 돌려보고 각 품질 기준의 통과율을 확인하는 겁니다:

```python
pilot_results = []

for i in range(50):
    oa = download_oa_pdf(application_number)
    claims_pre = download_publication(application_number)
    claims_post = download_registration(application_number)
    
    result = {
        "출원번호": application_number,
        
        # 기준 1: OA 상세도
        "oa_파싱_성공": oa에서 구성요소별 매핑 추출 가능한가,  # True/False
        "추출된_구성요소_수": N개,
        
        # 기준 2: diff 명확성  
        "diff_추출_성공": 자동 diff가 의미 있게 나오는가,  # True/False
        "diff_유형": "구성요소_추가" / "전면_재작성" / "변경없음",
        
        # 기준 3: 선행문헌 확보
        "인용문헌_한국특허_비율": M/N,  # 한국 특허면 확보 쉬움
        "인용문헌_전문_확보율": 실제로 텍스트 확보한 비율,
        
        # 기준 4: PDF 파싱
        "oa_파싱_깨짐": True/False,
        "청구항_파싱_깨짐": True/False,
        "선행문헌_표_있음": True/False,
        "표_파싱_성공": True/False,
    }
    pilot_results.append(result)

# 통과율 집계
print(f"OA 상세 매핑 있음: {상세_수}/50 = {상세_수/50:.0%}")
print(f"diff 자동 추출 가능: {diff_수}/50 = {diff_수/50:.0%}")
print(f"선행문헌 전문 확보: {확보_수}/50 = {확보_수/50:.0%}")
print(f"PDF 파싱 깨짐 없음: {깨끗_수}/50 = {깨끗_수/50:.0%}")
```

결과가 예를 들어 이렇게 나온다고 합시다:

```
OA 상세 매핑 있음:    60% (30/50)
diff 자동 추출 가능:   70% (35/50)
선행문헌 전문 확보:    80% (40/50)
PDF 파싱 깨짐 없음:   75% (37/50)

4개 기준 모두 통과:    40% (20/50)
```

이러면 **후보 5,000건에서 4개 기준 모두 통과하는 건 약 2,000건**이라는 추정이 나옵니다. 2,000건 목표와 딱 맞으면 전부 쓰면 되고, 부족하면 기준을 완화하거나 후보를 늘려야 합니다.

만약 "4개 모두 통과"가 20%밖에 안 되면, 후보 10,000건이 필요하다는 뜻이고, 이건 10년치로 범위를 넓히거나 품질 기준을 일부 완화해야 합니다. 예를 들어 "선행문헌이 일본 특허여도 기계번역으로 대체"하면 기준 3 통과율이 올라갑니다.

**이래서 50건 파일럿이 먼저인 겁니다.** 파일럿 없이 바로 2,000건 수집하면, 다 모은 다음에 "절반이 쓸 수 없는 데이터였다"는 걸 뒤늦게 알게 됩니다.
