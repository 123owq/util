분포보다 먼저 중요한 게 하나 있고, 그다음 분포 파악 방법을 알려드릴게요.

---

## 분포보다 먼저: 데이터 품질

2,000건의 균형 잡힌 저품질 데이터보다 **500건의 깨끗한 데이터가 낫습니다.** 품질이란 구체적으로:

- OA에 구성요소별 상세 매핑이 있는가 (뭉뚱그린 건 쓸모없음)
- 공개 청구항 → 등록 청구항 diff가 명확한가
- 인용 선행문헌 전문을 실제로 확보할 수 있는가
- PDF 파싱이 깨지지 않았는가 (표, 수식, 화학식 깨짐)

그래서 **먼저 50건 파일럿을 돌려서** "자동 파싱 정확도가 몇 %인가"를 확인한 뒤, 품질 기준을 정하고 나서 대량 수집으로 넘어가세요.

---

## 분포 파악 방법

### 1단계: 모집단 파악 (KIPRIS에서 무료로 가능)

KIPRIS 검색에서 이렇게 조건을 걸면 타이어 분야 전체 모집단 규모가 나옵니다:

```
검색 조건:
  IPC: B60C (타이어)
  출원일: 2015~2024 (최근 10년)
  행정상태: 전체 (등록/거절/취하 포함)
```

여기서 행정상태별 건수를 메모합니다:

```
등록:  약 N1건
거절:  약 N2건
취하:  약 N3건
심사중: 약 N4건
```

이 숫자만으로 "등록률"이 나오고, 이게 첫 번째 분포입니다.

### 2단계: 샘플 50건으로 세부 분포 파악

전체를 다 볼 수는 없으니까, **50건만 랜덤으로 열어서** 세부 분포를 수작업으로 집계합니다. KIPRIS에서 B60C 등록 특허를 50건 열고 중간서류철에서 의견제출통지서를 확인하면 됩니다.

체크할 항목:

```
[50건 샘플 체크시트]

# 1. 거절유형 분포
□ 진보성(§29②)만:          ___ 건
□ 신규성(§29①)만:          ___ 건
□ 기재불비(§42):           ___ 건
□ 진보성+기재불비 복합:      ___ 건
□ 기타:                    ___ 건

# 2. OA 횟수 분포
□ OA 1회 → 등록:           ___ 건
□ OA 2회 → 등록:           ___ 건
□ OA 3회+:                ___ 건

# 3. 기술 하위분야 분포
□ 트레드 패턴/구조:         ___ 건
□ 고무 조성물/배합:         ___ 건
□ 카카스/벨트 구조:         ___ 건
□ 제조 공정:               ___ 건
□ 기타:                    ___ 건

# 4. OA 상세도 분포 (가장 중요)
□ 구성요소별 단락 매핑 있음:  ___ 건  ← 학습에 쓸 수 있는 건
□ 뭉뚱그려 쓴 거절:         ___ 건  ← 버려야 할 건

# 5. 보정 유형 분포
□ 종속항 특징 통합:          ___ 건
□ 수치 범위 한정:           ___ 건
□ 구성요소 추가:            ___ 건
□ 보정 없이 의견서만:        ___ 건

# 6. 인용문헌 수
□ 1건:                     ___ 건
□ 2건:                     ___ 건
□ 3건+:                    ___ 건
```

이 50건 집계에 **3~4시간이면 충분**합니다. KIPRIS에서 중간서류철 열어서 OA 훑어보고 체크하는 건 건당 3~5분이면 됩니다.

### 3단계: 모집단 분포 추정

50건 샘플 결과가 예를 들어 이렇게 나왔다고 하면:

```
거절유형: 진보성 70%, 기재불비 15%, 신규성 5%, 복합 10%
OA 상세도: 상세 매핑 60%, 뭉뚱그림 40%
하위분야: 트레드 35%, 고무 30%, 구조 20%, 공정 15%
```

이 비율을 모집단에 적용하면:

```
모집단 등록 특허: 5,000건 (가정)
× OA 있는 비율 80%: 4,000건
× 상세 매핑 비율 60%: 2,400건  ← 학습에 쓸 수 있는 후보
```

2,400건 중 2,000건을 뽑으면 된다는 계산이 나옵니다.

### 4단계: 이 분포를 그대로 따를지, 조정할지 결정

**자연 분포를 그대로 따르면** 진보성 70%가 됩니다. 이게 실서비스에서의 비율과 일치하니까, 모델의 전체 정확도를 올리는 데는 이게 맞습니다.

**하지만 소수 클래스를 오버샘플링**하면 신규성·기재불비 케이스에서의 성능이 올라갑니다. 대신 전체 정확도는 약간 떨어질 수 있습니다.

| 전략 | 진보성 | 신규성 | 기재불비 | 복합 | 적합한 경우 |
|---|---|---|---|---|---|
| 자연 분포 | 1,400 | 100 | 300 | 200 | 전체 정확도 최대화 |
| 균형 조정 | 1,100 | 300 | 300 | 300 | 소수 클래스 성능도 중요 |
| 소수 클래스 집중 | 800 | 400 | 400 | 400 | 소수 클래스가 비즈니스상 중요 |

**권고는 균형 조정**입니다. 신규성 100건은 너무 적어서 해당 유형을 학습 못 합니다. 300건이면 최소한 패턴을 배울 수 있습니다.

## 이 전체 과정을 LLM으로 자동화

50건 수작업 말고, LLM을 써서 더 빠르게 할 수 있습니다:

```python
# OA PDF를 텍스트로 변환한 뒤
def classify_oa(oa_text):
    prompt = f"""
    다음 의견제출통지서를 분석하여 JSON으로 답하세요:
    
    {{
      "거절유형": ["진보성"/"신규성"/"기재불비" 중 해당하는 것 모두],
      "상세도": "상세" 또는 "간략",  // 구성요소별 단락 매핑이 있으면 "상세"
      "인용문헌수": 숫자,
      "기술분야": "트레드"/"고무조성물"/"구조"/"공정"/"기타"
    }}
    
    의견제출통지서:
    {oa_text}
    """
    return llm.generate(prompt)

# 300건 OA를 자동 분류 → 분포 집계 → 층화 추출
```

이러면 300건 분류가 1시간 안에 끝나고, 분포를 바로 파악할 수 있습니다. 그 중에서 "상세도: 상세"인 것만 필터링해서 2,000건을 채우면 됩니다.
